# 대시보드 데이터 요약 — 어떤 데이터가 어디서 오는가

JSON 구조 말고 **"어떤 지표가 어디서 오는지"** 만 간단히 정리.

---

## 1. GA4 (Google Analytics) — `/api/ga4`

사이트에서 발생한 **사용자 행동 이벤트** 데이터. 우리 사이트 `<script>`로 심은 gtag가 구글에 전송하고, 서버에서 Data API로 다시 끌어옵니다.

**오는 데이터:**
- 이벤트별 **발생 횟수** (events)
- 이벤트별 **순 사용자 수** (users)
- 전체 기간 **총 방문자 수**

**우리가 추적하는 이벤트:**
- `page_view` — 페이지 방문
- `cta_click` — "AI 생존유형 진단하기" 버튼 클릭
- `test_start` — 첫 문항 진입
- `test_complete` — 20문항 완료
- `result_view` — 결과 페이지 진입
- `openchat_click` — 단톡방 링크 클릭
- `ebook_click` — 전자책 CTA 클릭
- `share_click` — 공유 버튼 클릭

→ 이 수치들로 **퍼널 차트·전환율·2차 전환** 계산.

---

## 2. GA4 트래픽 분석 — `/api/ga4-traffic`

같은 GA4인데 **세션 차원의 분석 데이터**.

**오는 데이터:**
- **유입 소스/매체별** — 어디서 들어왔는지 (google/organic, facebook/cpc 등) + 사용자·세션·참여세션 수
- **국가별** — 방문자 국적 + 사용자·세션·참여 수
- **디바이스별** — mobile / desktop / tablet 구분
- **랜딩 페이지별** — 어떤 URL로 처음 들어왔는지
- **요약** — 총 방문·총 세션·참여 세션·한국 사용자 수

→ **봇 탐지**(해외 비율 높으면 의심), **세계지도 시각화**, **유입 소스 차트**에 사용.

## 3. Meta Ads — `/api/meta-ads`

페이스북/인스타 광고를 집행한 **캠페인별 성과 수치**. Meta 비즈니스 플랫폼에서 Graph API로 옴.

**오는 데이터:**
- **캠페인 이름** (`ai-mbti-...` 필터)
- **노출 수** (impressions) — 광고가 화면에 뜬 횟수
- **클릭 수** (clicks) — 광고를 누른 횟수
- **광고비** (spend) — 실제 지출 금액 (원화)
- **CTR** — 클릭률(%)
- **CPC** — 클릭당 비용(₩)
- **CPM** — 1,000회 노출당 비용(₩)
- **전환 수** — Pixel에 설정한 Lead 이벤트 발생 수
- **링크 클릭** (link_clicks) — 실제 외부 이동 수
- **랜딩 페이지 뷰** (page_views) — 랜딩 페이지 로드 완료 수

→ Meta Ads 뷰의 **KPI 카드·캠페인 테이블·전환 세분화** 구성.

---

## 4. Google Ads — `/api/google-ads`

구글 검색·디스플레이 광고의 **캠페인별 성과**. GAQL(SQL 비슷한 쿼리 언어)로 조회.

**오는 데이터:**
- **캠페인 이름** (이름에 `MBTI` 포함된 것만)
- **노출 수**
- **클릭 수**
- **광고비** — 단위 주의: `cost_micros` (원화 × 1,000,000로 옴, 나눠서 환산)
- **CTR** — 비율로 옴 (0.0266 = 2.66%, ×100 환산)
- **평균 CPC** — 마이크로 단위 (환산 필요)
- **전환 수** — Google Ads에 등록한 전환 액션 기준

→ Google Ads 뷰에서 Meta와 동일한 UI 포맷으로 표시. 단위 환산·합계는 서버에서 처리 후 프론트에 전달.

**특이사항:** Google Ads는 **1시간짜리 Access Token**이라 매 요청마다 Refresh Token으로 재발급받은 뒤 데이터를 조회.

## 5. A/B 테스트 — `/api/ab-test`

**우리 Supabase DB에 저장된 자체 이벤트**. GA4와 별개로 랜딩 A/B 테스트를 위해 직접 저장.

**오는 데이터:** 변형별(v1·v3·v4) 단계 수치
- 방문 수 (pageView)
- CTA 클릭 (ctaClick)
- 테스트 시작 (testStart)
- 테스트 완료 (testComplete)
- 결과 확인 (resultView)
- 단톡 클릭 / 전자책 클릭 / 공유 클릭

**변형 구분 기준:** 브라우저에서 이벤트를 기록할 때 `metadata.ab_variant` 필드에 `v1`/`v3`/`v4` 넣어둠 → 서버가 그걸 기준으로 필터해서 각각 집계.

→ A/B 테스트 뷰의 **변형별 퍼널·전환율 비교표·승자 배너**에 사용.

## 6. 메타코드 전자책 수강생 — `/api/metacode-ebooks`

**메타코드 공식 사이트의 실제 수강 등록 인원**. 메타코드 공개 API를 Basic Auth로 호출.

**오는 데이터:** 전자책 4종 각각에 대해
- 전자책 ID
- 제목
- **실제 수강 등록 인원** (`cnt_join`)

**필터 기준:** 슈퍼 이벤트 메뉴(`menu2_idx=30656`) 안의 EP_IDX 4개만 (27433 / 27427 / 27426 / 27422).

→ 전자책 뷰의 **총 수강생 숫자 + 개별 전자책 카드** 구성.

**주의:** GA4의 `ebook_click` (버튼 클릭 수)과 이 수치(실제 등록 완료 수)는 **다른 의미**. 전자는 "관심 표현", 후자는 "최종 전환".

## 한눈에 요약 — 어떤 수치가 어디서?

| 대시보드 수치 | 출처 | 비고 |
|--------------|------|------|
| 방문자 수, CTA 클릭, 테스트 완료, 결과 확인 | GA4 | `users` 기준 (중복 제거) |
| 2차 전환 (단톡/전자책/공유 클릭) | GA4 | 이벤트 users 합산 |
| 국가·디바이스·유입 소스 | GA4 트래픽 | 세션 차원 |
| Meta 광고비·CTR·CPC·전환 | Meta Graph API | 캠페인 이름에 `ai-mbti` 포함 |
| Google 광고비·CTR·CPC·전환 | Google Ads API | 캠페인 이름에 `MBTI` 포함, 마이크로 단위 환산 |
| A/B 변형별 퍼널 (v1/v3/v4) | Supabase events | 우리 DB, 이벤트 기록 시 variant 태깅 |
| 전자책 실수강생 | 메타코드 API | 4개 ID만 필터 |
| 유효 CPA (대시보드 계산) | Meta + Google 광고비 ÷ 2차 전환 | 서버에서 계산 |

## 데이터 신선도

| 출처 | 반영 지연 | 비고 |
|------|-----------|------|
| GA4 | 최대 24~48시간 | 당일 데이터는 GA4 쪽에서 집계 중일 수 있음 |
| GA4 트래픽 | 최대 24~48시간 | 동일 |
| Meta Ads | 거의 실시간 (수분) | API 호출 때 가져옴 |
| Google Ads | 거의 실시간 (수분) | 동일 |
| Supabase events | 실시간 | 브라우저에서 저장되는 즉시 |
| 메타코드 전자책 | 실시간 | API 호출 때 현재 값 |
