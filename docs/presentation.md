# AImBTI 발표 자료

> 대상: 대표님 + 동료 / 소요 시간: 5~7분

---

## 1. 한 줄 소개

**"내가 AI에 대체될까?" — MBTI 형식 진단으로 답하고, 부트캠프 상담으로 연결하는 바이럴 리드 수집 퍼널**

---

## 2. 왜 만들었나

- "AI 대체 위협"은 누구나 한번쯤 검색하는 키워드
- 진단 결과를 공유하게 만들면 → **광고비 없이 바이럴** 가능
- 위협감을 느낀 사람이 → **부트캠프 상담 신청(리드)** 으로 이어짐

---

## 3. 서비스 플로우

```
랜딩 (/)
  → 테스트 시작 (/test)
      → 20문항 답변
          → 분석 애니메이션
              → 결과 페이지 (/result/[id])
                  → 유형 확인 + AI 대체 점수
                      → 무료 상담 신청 (이름 + 전화번호)
                          → 부트캠프 URL 오픈 + 쿠폰 발급
                              → 카카오/링크 공유 → 바이럴
```

---

## 4. 유형 시스템

### 4축 기반 16유형

20문항을 4개 파트로 나눠 각 파트에서 다수결로 한 글자를 결정 → 4글자 코드 산출

| 파트 | 축 | 의미 |
|---|---|---|
| A | **H** vs **T** | 독립형(H) vs 협업형(T) — 업무 방식 |
| B | **A** vs **S** | AI 적극활용(A) vs 신중형(S) — AI 활용도 |
| C | **L** vs **C** | 논리/분석형(L) vs 창의/감성형(C) — 강점 |
| D | **F** vs **P** | 속도우선(F) vs 완성도우선(P) — 실행 방식 |

> 예: 독립 + AI활용 + 논리 + 속도우선 → `HALF`

### 유형별 부트캠프 추천

| 조건 | 추천 부트캠프 | 해당 유형 |
|---|---|---|
| AI 적극 활용 + 논리형 | **AI 서비스 개발자** | HALF, HALP, TALF, TALP |
| AI 적극 활용 + 창의형 | **AI LLM** | HACF, HACP, TACF, TACP |
| AI 신중 + 논리형 + 독립 | **데이터 엔지니어** | HSLF, HSLP |
| AI 신중 + 논리형 + 협업 | **데이터 분석** | TSLF, TSLP |
| AI 신중 + 창의형 | **데이터 분석** | HSCF, HSCP, TSCF, TSCP |

진단 결과에서 AI 대체 점수 + 추천 부트캠프를 함께 보여줘서 자연스럽게 상담으로 유도.

## 7. 데이터 분석 — GA4 + Clarity

### GA4: 퍼널 단계별 이벤트 추적

GA4 연동은 간단하게 `gtag` 스크립트를 `<head>`에 삽입하고,  
필요한 시점마다 `window.gtag('event', ...)` 한 줄로 이벤트를 쏘는 방식.

**데이터 흐름**
```
사용자 브라우저
  → gtag('event', ...) 호출
      → Google 서버 (analytics.google.com) 로 전송
          → GA4 대시보드 (측정 ID: G-XXXXXXXX 기준으로 집계)
              → 실시간 / 이벤트 / 유입경로 보고서에서 확인 가능
```

- 서버를 거치지 않고 **브라우저 → Google 직접 전송** (클라이언트 사이드)
- 별도 백엔드 없이 `gtag` 스크립트 한 줄로 모든 이벤트 수집 가능
- GA4 대시보드 > 이벤트 메뉴에서 이벤트명·파라미터별로 집계 확인

```js
// 예시: 테스트 완료 시
gtag('event', 'test_complete', { type_code: 'HALF' })

// 예시: 리드 제출 시
gtag('event', 'lead_submit', { source: 'bootcamp_modal', type_code: 'HALF' })
```

| 이벤트 | 발생 시점 |
|---|---|
| `cta_click` | 랜딩 CTA 버튼 클릭 |
| `test_start` | 테스트 첫 문항 진입 |
| `test_complete` | 결과 페이지 도달 |
| `share_click` | 카카오/링크 공유 클릭 |
| `lead_submit` | 상담 신청 완료 |
| `bootcamp_click` | 부트캠프 URL 클릭 |
| `coupon_unlock` | 쿠폰 노출 |

페이지 이동마다 `page_title`도 함께 전송 → GA에서 페이지별 구분 가능.

### Microsoft Clarity: 세션 녹화 + 히트맵 (연동 중)

- GA4와 Clarity **연동 진행 중** → 완료 시 GA4 세그먼트 기준으로 Clarity 녹화 필터링 가능
- 실제 사용자가 어디서 막히는지, 어디를 클릭하는지 세션 영상으로 확인
- 리드 전환 안 된 사용자 행동 패턴 분석에 활용 예정

## 10. 다음 단계 — 메타코드 사이트 연결 방향 고민 중

리드를 확보한 이후, 어떻게 전환 가치를 높일지 3가지 방향을 검토 중.

### 옵션 1. 카카오톡 채널에서 전자책 구매
- 카카오 채널에서 결제 → Google Sheets에서 구매 정보 관리
- 장점: 카카오 기반 자연스러운 플로우, 별도 회원가입 없음
- 고민: 결제 시스템 연동 복잡도, Sheets 자동화 필요

### 옵션 2. 이름 + 전화번호 + 이메일 수집 → 이메일로 전자책 전송
- 지금 리드 수집 플로우에 이메일 필드 추가
- 제출 시 자동으로 이메일 발송 (전자책 첨부 또는 다운로드 링크)
- 장점: 구현 단순, 이메일 DB 확보
- 고민: 이메일 발송 서버 필요 (Resend / SendGrid 등)

### 옵션 3. 메타코드 사이트 회원가입 → 전자책 제공
- 진단 결과 페이지 CTA를 "회원가입하면 전자책 무료 제공"으로 변경
- metacodes.co.kr 회원 DB로 유입 → 장기 리텐션 가능
- 장점: 가장 강력한 전환 (회원 확보), 맞춤 콘텐츠 제공 가능
- 고민: 회원가입 마찰 높음, metacodes 사이트 연동 개발 필요
