## 4. Lead 확인 방법

### 4-1. Meta 이벤트 관리자 (실시간 테스트)
https://business.facebook.com/events_manager2/list/pixel/4172135629693592/overview

"테스트 이벤트" 탭 → 브라우저 URL 입력 → 결과 페이지에서 이북/오픈챗 클릭 시 Lead 발화 확인.

### 4-2. Chrome 확장 "Meta Pixel Helper"
설치 후 결과 페이지 접속 → 아이콘에 `PageView`, `Lead` 카운트 표시.

### 4-3. 브라우저 콘솔 (수동 트리거)
```js
fbq('track', 'Lead', { content_name: 'manual-test', type_code: 'AHLF' })
```
DevTools → Network 탭에서 `facebook.com/tr/?...ev=Lead` 요청 확인.

### 4-4. Meta Ads API (캠페인별 집계)
```bash
source <(grep -E "^META_ADS_" .env | sed 's/^/export /')
ACCOUNT_ID="act_${META_ADS_ACCOUNT_ID#act_}"

curl -sS "https://graph.facebook.com/v19.0/${ACCOUNT_ID}/insights\
?level=campaign\
&date_preset=last_7d\
&fields=campaign_name,impressions,clicks,spend,actions\
&access_token=${META_ADS_ACCESS_TOKEN}"
```
- `actions[]` 중 `action_type == "lead"` 또는 `offsite_conversion.fb_pixel_lead` 가 Lead 카운트.
- `level=adset` / `level=ad` 로 바꾸면 소재 단위까지 확인 가능.

### 4-5. Pixel 수집 통계 (전체 사이트 기준)
```bash
PIXEL_ID="4172135629693592"
curl -sS "https://graph.facebook.com/v19.0/${PIXEL_ID}/stats\
?start_time=$(date -d '1 day ago' +%s)\
&end_time=$(date +%s)\
&aggregation=event\
&access_token=${META_ADS_ACCESS_TOKEN}"
```

### 4-6. Supabase `events` 테이블 (자체 이중화)
픽셀과 별개로 `src/lib/track.ts`가 기록:
```sql
select event, count(*)
from events
where event in ('ebook_click','openchat_click')
  and created_at > now() - interval '7 days'
group by event;
```
대시보드: http://localhost:3000/dashboard3
