# A/B 테스트 실행 가이드 (Meta·Google Ads)

랜딩 변형 3개 중 **어느 게 가장 잘 먹히는지** 광고 예산으로 실험하는 방법.

---

## 0. 미리 준비된 것

| 항목 | 상태 | 경로 |
|------|------|------|
| 변형 페이지 3개 | ✅ 완료 | `/preview/v1` `/preview/v3` `/preview/v4` |
| 이벤트 태깅 (`ab_variant`) | ✅ 완료 | 랜딩 진입 시 자동 |
| 대시보드 A/B 테스트 탭 | ✅ 완료 | `/dashboard3` → A/B 테스트 |
| 승자 자동 판정 | ✅ 완료 | E2E 전환율 기준 |

→ **광고 트래픽만 3개 URL로 나눠 보내면 나머지는 자동 집계됨.**

## 1. 변형 컨셉 정리

| 변형 | 컨셉 | 헤드라인 예 |
|------|------|-----------|
| `v1` | 공포소구 | "AI가 당신 월급을 노리고 있습니다" |

## 2. Meta Ads 세팅

### 2-1. 캠페인 복제 방식 (추천)

**Business Manager → 광고 관리자 → 캠페인 생성**

```
캠페인 A: "ai-mbti-v1-fear"
  → 최종 URL: https://mcodegc.com/preview/v1?utm_source=meta&utm_campaign=v1-fear

캠페인 B: "ai-mbti-v3-social"
  → 최종 URL: https://mcodegc.com/preview/v3?utm_source=meta&utm_campaign=v3-social

캠페인 C: "ai-mbti-v4-simple"
  → 최종 URL: https://mcodegc.com/preview/v4?utm_source=meta&utm_campaign=v4-simple
```

**3개 캠페인 동일 조건:**
- **목표:** 전환 (Lead 최적화)
- **타겟팅:** 만 25~45세 / 수도권 / 직장인·자기계발 관심사
- **일 예산:** 각 10,000원 (3개 합 30,000/일)
- **집행 기간:** 동시 시작, 최소 14일
- **입찰 전략:** 최저 비용 (Lowest Cost)

**광고 소재 원칙:**
- **랜딩 메시지와 일치**시키기 (공포 랜딩엔 공포 광고, 사회적 증거 랜딩엔 수치 강조 광고)
- 이미지·카피만 다르게, 다른 요소(CTA 문구·포맷)는 동일

### 2-2. 1개 캠페인 안에 광고 세트 3개 (정밀한 방법)

Meta가 예산 최적화로 특정 세트에 예산 몰아주는 걸 방지하려면:
- **CBO 끄고 ABO** (Ad Set Budget Optimization) 선택
- 세트별 예산 고정 (10,000원씩)
- 같은 오디언스 그룹 사용

## 3. Google Ads 세팅

### 3-1. 검색 광고 (Search Ads)

**새 캠페인 3개** 만들고 **동일 키워드 그룹** 사용:

```
캠페인 A: "MBTI-v1-Search"
  → 최종 URL: /preview/v1?utm_source=google&utm_campaign=v1-search

캠페인 B: "MBTI-v3-Search"
  → 최종 URL: /preview/v3?utm_source=google&utm_campaign=v3-search

캠페인 C: "MBTI-v4-Search"
  → 최종 URL: /preview/v4?utm_source=google&utm_campaign=v4-search
```

**동일 설정:**
- **키워드:** "AI MBTI 테스트", "AI 대체 직업", "ChatGPT 일자리" 등 (3개 캠페인 모두 동일)
- **입찰 전략:** 수동 CPC 또는 타겟 CPA
- **일 예산:** 10,000원씩
- **지역 타겟:** 🇰🇷 대한민국만 (!! 매우 중요 — 해외 유입 봇 차단)
- **디스플레이 네트워크(GDN):** **끄기** (정밀한 검색 테스트 위해)

### 3-2. UTM 필수

Google Ads는 자동 태깅(`gclid`)이 있지만, 추가로 UTM 파라미터 붙이면 대시보드 트래픽 분석에서 구분이 쉬워짐.

## 4. 결과 확인 (대시보드)

### 4-1. 실시간 확인 위치

- **A/B 테스트 탭**: 변형별 퍼널 자동 집계
- **Meta Ads 탭**: 캠페인 이름(`ai-mbti-v1-fear` 등)에 `v1/v3/v4` 포함되어 있어 테이블에서 식별
- **Google Ads 탭**: 캠페인 이름(`MBTI-v1-Search`)으로 식별

### 4-2. 주요 비교 지표

| 지표 | 의미 | 어디서 보는가 |
|------|------|--------------|
| **광고 CTR** | 광고 소재 반응 | Meta/Google Ads 탭 |
| **랜딩 CTR** (방문→CTA) | 랜딩 헤드라인·오퍼 효과 | A/B 테스트 탭 |
| **E2E 전환율** (방문→결과) | 전체 퍼널 효과 — 승자 판정 기준 | A/B 테스트 탭 |
| **CPA** (광고비÷2차 전환) | 종합 효율 — 최종 의사결정 | 종합 뷰 |

---

## 5. 판정 규칙

### 5-1. 최소 샘플

각 변형당 **방문 300명 이상** 확보 후 판단.
- 100명 미만: 판단 불가 (노이즈 지배)
- 100~300명: 감 잡기만
- 300+: 신뢰 가능

### 5-2. 승자 조건

**E2E 전환율** 기준:
- 승자 - 2위 = **+2%p 이상** → 확실한 승자
- +0.5~2%p → 더 돌려보고 판단
- +0.5%p 미만 → 동점, 다른 지표(CPA 등)로 결정

### 5-3. 중단 금지

- 초반 1~3일은 소재·알고리즘이 학습 중이라 **노이즈가 큼**
- **최소 7일, 권장 14일**까지는 중단 없이 돌리기
- "이건 분명히 이겼다" 싶어도 기간 다 채우기 — 통계적 안정성 확보

## 6. 권장 진행 순서 (토너먼트)

3개 동시보다 2개씩 비교가 빠름:

```
Round 1 (2주)
  v1 vs v3
    → 승자: v3 (가정)

Round 2 (2주)
  v3 vs v4
    → 최종 승자: v4 (가정)

→ v4를 메인 랜딩(/)에 적용
```

1라운드에 소재·카피 같은 차원에서 승자를 정한 다음, 2라운드는 다른 차원을 테스트하는 식.

## 7. 승자 확정 후 액션

1. **승자 변형을 메인 랜딩으로 복사**
   - `src/app/preview/v3/page.tsx` 내용 → `src/app/page.tsx`에 덮어쓰기
   - 또는 `middleware.ts`에서 `/` 접근 시 승자 URL로 리다이렉트
2. **패배 변형은 광고 중단** (Meta/Google 해당 캠페인 Pause)
3. **새 변형 만들어서 다음 라운드**
   - 승자 기준으로 한 요소만 바꿔서 v5, v6 생성
   - 같은 방식으로 테스트

## 9. 실전 팁

| 상황 | 대응 |
|------|------|
| 1일차에 한쪽이 0명 | 광고 미승인 or URL 오타 점검 |
| 해외 방문이 많음 | 광고 지역 타겟이 한국으로 제한됐는지 확인 |
| CPC가 너무 비쌈 | 광고 소재 교체 (CTR 올리기) or 타겟 재검토 |
| CTR은 좋은데 전환 없음 | 랜딩과 광고 메시지 불일치, 랜딩 카피 개선 |
| 3개 다 비슷 | 차별점이 약하다는 뜻 — 더 극단적 변형 만들기 |

## 용어 정리

| 약어 | 풀이 | 설명 |
|------|------|------|
| CTR | Click-Through Rate | 노출 대비 클릭률 |
| CPC | Cost Per Click | 클릭 1회당 비용 |
| CPA | Cost Per Acquisition | 전환 1건당 비용 |
| CVR | Conversion Rate | 클릭 대비 전환율 |
| E2E | End-to-End | 퍼널 처음~끝 전체 전환율 |
| ABO | Ad Set Budget Optimization | 세트별 예산 고정 |
| CBO | Campaign Budget Optimization | 캠페인 통합 예산 (Meta 알고리즘이 분배) |
| UTM | Urchin Tracking Module | URL 뒤에 붙이는 추적 파라미터 |
| GDN | Google Display Network | 구글 배너 광고 네트워크 |
